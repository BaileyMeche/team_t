## Strategy Overview

This project forecasts **which S&P 500 stocks will move most sharply following their quarterly earnings announcements**, then trades those predictions through **delta-hedged long call options** to generate direction-agnostic gamma exposure. The signal is generated by an LSTM trained exclusively on company financial changes and market microstructure features — using strictly point-in-time data, with no lookahead bias at any stage.

The strategy exploits the empirically documented **post-earnings announcement drift (PEAD)**: prices continue moving in the direction of the earnings surprise for days to weeks after the announcement, and the magnitude of this drift is predictable from pre-announcement fundamental and market signals.

---

## Core Hypothesis

Company financials arrive with a ~45-day filing lag, but prices move daily. This temporal mismatch creates a window: an LSTM that reads 60-day sequences of fundamental changes and market signals can rank stocks by expected post-announcement move magnitude. Long-gamma positions via delta-hedged calls monetize realized volatility in the stocks that move most, regardless of direction.

## Universe Construction

The tradeable universe is **not fixed** — it is rebuilt annually to stay current with S&P 500 membership, avoid survivorship bias, and maintain options liquidity.

**Annual rebalance procedure (first trading day after May 15, 2006–2024):**

1. Start from full S&P 500 constituent list for that year
2. Filter: must have OptionMetrics coverage, ≥2 years price history, stable rolling beta, adequate average daily volume
3. Exclude Financials and Utilities (accounting ratios are structurally distorted)
4. Cap at 8 stocks per GICS sector to enforce diversification
5. Apply turnover buffer: incumbent stocks receive preference to limit unnecessary churn
6. Target: **30 stocks per rebalance year**

**Result:** 19 annual rebalances · 30 names per year · **58 unique tickers** across 2010–2024

## Feature Engineering

The model uses **13 signals** organized in two groups, all standardized cross-sectionally each day (z-scored within the universe) so the model compares relative rank, not absolute level.

### Group 1 — Company Financial Changes (8 features)

These measure the *direction and rate of change* of fundamentals quarter-over-quarter, not static levels.

| Feature | Construction |
|---|---|
| ROE change | Quarter-over-quarter Δ return on equity |
| Operating margin change | Quarter-over-quarter Δ EBIT/revenue |
| Leverage change | Quarter-over-quarter Δ total debt / equity |
| EPS growth | Quarter-over-quarter Δ diluted EPS |
| Book value growth | Quarter-over-quarter Δ book value per share |
| EPS acceleration | Second difference of EPS (change-in-change) |
| Operating leverage surprise | Δ operating margin relative to trailing-4Q mean |
| Book value acceleration | Second difference of book value per share |

### Group 2 — Market Context (5 features)

These capture the pre-announcement price environment and trading regime.

| Feature | Construction |
|---|---|
| Recent stock return | Cumulative return over trailing 60 calendar days |
| Rolling beta | OLS β vs. S&P 500 over trailing 60 trading days |
| Abnormal volume | Trailing 5-day average volume / trailing 60-day average |
| Price reaction speed | Fraction of 60-day return realized in trailing 5 days |
| Volatility ratio | 5-day realized vol / 60-day realized vol (regime signal) |

All features are lagged by the 45-day filing buffer before entering any model or feature window.

## Model: Walk-Forward LSTM

### Architecture

- **Input:** sequences of shape `(60 days × 13 features)` per stock per date
- **Output:** predicted next-day return (regression target, not classification)
- **Training set:** all 58 stocks trained jointly — the model learns cross-sectional patterns
- **Total training sequences:** 124,824

### Walk-Forward Validation (Expanding Window)

The model is **retrained from scratch each calendar year** using only data available prior to that year. This eliminates any form of future leakage from model weights.

| Training Window | Prediction Year |
|---|---|
| 2006–2009 | 2010 |
| 2006–2010 | 2011 |
| 2006–2011 | 2012 |
| ⋯ | ⋯ |
| 2006–2023 | 2024 |

**15 out of 15 annual folds trained successfully.**  
**109,066 strictly out-of-sample predictions produced across 2010–2024.**

## Signal Generation & Trade Execution

### Step 1 — Earnings-Triggered Ranking

- On each day where ≥1 universe stock has an earnings announcement the following morning, the model generates return predictions for all active universe stocks
- Stocks are ranked by |predicted return| (magnitude, not direction — we are long gamma)
- **Top K = 3** stocks selected per event; tie-broken by trailing average daily dollar volume

### Step 2 — Entry Timing

- Prediction made the **day before** the earnings announcement (avoiding inflated pre-announcement IV)
- **Entry the morning after** the announcement (post-gap, after initial volatility settles)
- 458 qualifying earnings events across 2010–2024

### Step 3 — Delta-Hedged Long Call

For each selected stock:

1. **Buy** the nearest-to-ATM call option with 30–45 days to expiration (sourced from OptionMetrics)
2. **Short** `Δ × 100` shares of the underlying simultaneously to zero out directional (delta) exposure
3. **Rebalance** the stock short daily as delta drifts with stock price movement
4. **Hold** up to 3 trading sessions, exit early if: (a) stock drops from top-3 ranking, or (b) stop-loss triggers

## Backtest Design

| Parameter | Value |
|---|---|
| Starting capital | $1,000,000 |
| Evaluation period | January 2010 – December 2024 |
| Capital per trade | ~1% of portfolio (~$10,000) |
| Maximum hold period | 3 trading sessions |
| Stop-loss threshold | 5% of entry premium cost |
| Entry signal threshold | Predicted return ≥ 0.10% (filters pure noise) |
| Exit trigger | Stock exits top-3 LSTM ranking |
| Benchmark | SPY total return (buy-and-hold) |

Transaction costs are modeled explicitly via an OOA (order-of-approximation) framework averaging approximately 499 bps round-trip, contributing ~$7.4K in total execution drag over the sample.

## Risk Controls

### Position-Level

- Maximum position size: **1% of portfolio capital**
- Hard stop-loss: **5% of entry premium cost**
- Maximum hold: **3 trading sessions**
- Automatic exit if stock's LSTM rank deteriorates below top-3
- Entry gated by minimum prediction threshold (0.1%)

### Portfolio-Level

- Daily total exposure tracking (peak: $109K, 10.9% of NAV)
- Drawdown alert tiers at −5%, −10%, −15%
- Daily VaR and expected shortfall computation
- Sector concentration monitoring
- Volatility-spike detection

**319 risk events flagged for review across the full sample period. Zero hard limit breaches.**

All risk metrics are computed from actual executed trade P&L — not hypothetical signal P&L.

## Point-in-Time (PIT) Discipline

Preventing lookahead bias is the most critical correctness property of this system. Three independent safeguards are enforced:

**1. 45-Day Filing Lag**  
All fundamental data is shifted forward by 45 calendar days from quarter-end before any merge or feature construction. A December 31 quarter-end yields data available no earlier than February 14.

**2. Expanding Walk-Forward Retraining**  
The LSTM is retrained from scratch each year using only data from strictly prior years. The model weights for predicting 2015 have never seen a single data point from 2015 or later.

**3. Automated PIT Validation**  
`validate_point_in_time_panel(...)` in `data_utils.py` performs a systematic audit of every row in the merged panel, verifying that `availability_date ≤ prediction_date` for all fundamental fields. The function raises immediately on any violation — it does not warn, it crashes. This ensures no silent lookahead can survive into the feature or tensor pipeline.

## Next Steps

The research framework is production-grade and extensible. The primary performance limitation is signal quality, not infrastructure. Highest-priority extensions:

| Direction | Description |
|---|---|
| Universe expansion | Scale from 30 to 100+ stocks per year; more earnings events → better law-of-large-numbers properties |
| Longer holding horizons | Test 5-, 10-, and 20-day holds to allow PEAD more time to realize |
| Signal diagnostics | Compute IC (information coefficient), rank correlation, and Brier scores on LSTM predictions before options overlay |
| Equity-only baseline | Run identical ranking signal with plain stock positions to isolate model alpha from options structure gamma |
| Conviction-weighted sizing | Replace fixed 1% allocation with Kelly-fraction or vol-targeting position sizes scaled by LSTM confidence |
| Stress testing | Explicit scenario analysis through 2008–2009, March 2020, and 2022 rate shock regimes |
| Alternative architectures | Benchmark LSTM against gradient-boosted trees (XGBoost/LightGBM) and transformer-based sequence models on identical walk-forward folds |

## Setup & Reproducibility

### Requirements

```bash
pip install -r requirements.txt
```

### Data

`data/PRICES.csv` must be present before running any notebook cells. This file is the **deterministic data contract** for the pipeline — there is no fallback filename. The system hard-crashes on absence to prevent silent data mismatches.

Zacks fundamental data is fetched via the Nasdaq Data Link API. Credentials must be set in `.env`:

```
NASDAQ_DATA_LINK_API_KEY=your_key_here
```

WRDS OptionMetrics access requires institutional subscription credentials (available via the University of Chicago WRDS portal).

### Running the Notebook

```bash
jupyter notebook notebooks/lstm.ipynb
```

Pipeline checkpoint cells will validate PIT correctness automatically. Any validation failure raises immediately with the offending row(s) identified.
